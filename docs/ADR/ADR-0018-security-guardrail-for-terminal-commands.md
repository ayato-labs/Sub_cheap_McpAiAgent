# ADR-0018: ターミナルコマンド実行に対するセキュリティガードレールの導入

- **日付**: 2026年6月20日
- **ステータス**: 承認済み
- **決定者**: ayato-labs、Geminiエージェント
- **更新**: 本決定により ADR-0013 の「ノーガード」戦略を一部変更し、セキュリティガードレールを導入します。

## 背景
ADR-0013で導入された `execute_command` ツールは、利便性と速度を優先し、セキュリティ上の制限を一切設けない「ノーガード」戦略をとっていました。しかし、メインAI（Architect）がハルシネーション（幻覚）を起こし、意図せず破壊的なコマンド（ディレクトリ削除やシステム設定変更など）を発行した場合、何の制限もなくホストOS上で実行されてしまうリスクが顕在化しました。

## 決定事項
安全性と利便性のバランスをとるため、`execute_command` ツールにおいて、コマンド実行前にブラックリストを用いた簡易的な静的解析を行い、破壊的なコマンドをインターセプトするガードレールを導入します。

1. **ブラックリストの導入**:
   - `format`, `diskpart`, `reg delete`, `rm -rf /` などの非常に破壊的なコマンドを検知するブラックリストを実装します。

2. **静的解析によるインターセプト**:
   - ツール実行の冒頭でコマンド文字列を解析し、マッチした場合は実行を即座に中止し、セキュリティエラーを返します。

3. **セキュリティと利便性の維持**:
   - 複雑なサンドボックス環境（Dockerなど）の導入は、システム構成の複雑化とメンテナンスコストの増大を招くため、今回は採用しません。最低限の静的チェックによって「破壊的コマンドの誤発行による重大なシステム被害」を防ぐことを主眼とします。

## 影響
### メリット
- **破壊的被害の防止**: 誤ったコマンド実行による重大なシステム障害やデータ消失のリスクが軽減されます。
- **安全性への配慮**: AIによる実行の安全性に対して、最低限の防衛ラインが確保されます。

### デメリット／リスク
- **防衛力の限界**: 静的解析によるブラックリスト方式であるため、巧妙なコマンドや未知の破壊的コマンドを全て防ぐことはできません。あくまでAIのハルシネーションによる誤操作に対する最初の防衛線です。
- **管理コスト**: 今後、新しい破壊的コマンドが確認された場合、適宜ブラックリストを更新する必要があります。
