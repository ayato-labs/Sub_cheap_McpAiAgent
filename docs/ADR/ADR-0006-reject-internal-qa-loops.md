# ADR-0006: Rejection of Internal QA Loops (Semgrep)

- **Date**: 2026-06-15
- **Status**: Accepted
- **Deciders**: ayato-labs (User), Gemini CLI (Agent)

## Context
To improve the quality of code generated by inexpensive sub-LLMs, we considered integrating a local Quality Assurance (QA) loop within the MCP server using Semgrep. The proposed workflow was: Generate Code -> Run Semgrep -> If errors exist, send errors back to the sub-LLM to fix -> Return final code to Main AI. To save development costs, it was suggested to cap this loop at an arbitrary number (e.g., 3 retries).

## Decision
**Reject** the integration of Semgrep and any internal "generate-and-fix" retry loops within the sub-agent. 

## Rationale
1. **Violation of Core Philosophy**: The foundational philosophy of this project (defined in Conceptual Requirements 3.1) is that the sub-agent is a "stateless drafting factory." It must not perform complex autonomous reasoning, judgment, or self-correction (Agentic Workflows). QA, evaluation, and iteration are strictly the responsibilities of the Main AI or the CI/CD pipeline.
2. **Arbitrary Constraints**: The suggestion to use a hardcoded "3-loop limit" to save development costs highlighted that the feature itself was not essential. If an architectural component requires arbitrary constraints simply to be viable, it often indicates a flaw in the design boundary.
3. **Latency and Complexity**: Implementing an internal loop multiplies the latency of the sub-LLM API calls. It also requires managing language-specific rulesets for Semgrep and parsing its output, turning a simple, elegant single-file MCP server into a brittle, complex orchestration engine.
4. **Separation of Concerns**: The sub-agent's job is generation based on provided context. The Main AI's job is review and orchestration. Mixing these roles degrades the predictability and speed of the system.

## Consequences
### Positive
- The MCP server maintains its extreme simplicity, remaining a true "stateless function."
- Predictable and fast latency for the Main AI, as the sub-agent only ever makes a single generation pass per request.
- No heavy external dependencies (like Semgrep binaries) required for users to install.

### Negative / Risks
- The Main AI may occasionally receive code with syntax errors or security flaws, requiring the Main AI to spend its own tokens to review and issue a correction command. This is accepted as a necessary trade-off for architectural purity.
